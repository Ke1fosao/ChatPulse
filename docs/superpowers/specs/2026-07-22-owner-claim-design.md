# ChatPulse Owner Claim — Design Specification

## Goal

Закріпити один Telegram-акаунт як глобального власника ChatPulse. Власник отримує адміністративні можливості в боті та Mini App навіть тоді, коли Telegram не видає йому роль адміністратора конкретної групи.

## Approved account

Початкове закріплення доступне лише поточному власнику username `@veheblya`.

Username використовується лише під час одноразової активації. Після успішної активації ChatPulse зберігає незмінний Telegram user ID і надалі перевіряє права тільки за ID.

## Claim flow

1. Користувач відкриває приватний чат із ботом.
2. Надсилає `/claimadmin`.
3. Backend нормалізує username без `@` та без урахування регістру.
4. Якщо username не `veheblya`, команда відхиляється.
5. Якщо власника ще немає, створюється єдиний запис із Telegram user ID, username і часом активації.
6. Якщо запис уже належить тому самому ID, бот повідомляє, що акаунт уже є власником.
7. Якщо запис належить іншому ID, повторна активація блокується.

## Persistence

Нова таблиця `bot_owner` містить один логічний запис:

- `owner_key = 'primary'` — первинний ключ-синглтон;
- `telegram_user_id` — незмінний Telegram ID власника;
- `claimed_username` — username у момент активації;
- `claimed_at` — час активації.

Таблиця не має команди видалення або перепризначення через Telegram. Це запобігає захопленню ролі після зміни username.

## Permission integration

Глобальний власник проходить admin-check у двох контурах:

- групові команди `/settings`, `/setreporttime`, `/resetstats` і callback-кнопки;
- Mini App admin API для зміни налаштувань та скидання статистики.

Перевірка членства й доступу до самої групи не скасовується. Власник не отримує приховану статистику груп, учасником яких він не є.

## Security properties

- активація лише в приватному чаті;
- лише username `@veheblya` може виконати первинний claim;
- після claim username більше не є джерелом авторизації;
- один singleton-запис у базі;
- конкурентні спроби claim безпечно завершуються одним власником;
- RLS увімкнено, прямий доступ `anon` та `authenticated` до таблиці заборонено;
- секрети, токени й тексти повідомлень не зберігаються.

## Tests

- правильний username може закріпити роль;
- неправильний username відхиляється;
- повторний claim тим самим ID є ідемпотентним;
- інший ID не може перепризначити роль;
- owner bypass працює для bot admin-check;
- owner bypass працює для Mini App admin-check;
- звичайні користувачі й надалі перевіряються через Telegram `getChatMember`.
