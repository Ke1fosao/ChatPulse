# ChatPulse Mini App Dashboard — Design Specification

## 1. Мета

Створити повноцінний Telegram Mini App для ChatPulse, який об’єднує:

- глобальний профіль користувача в ChatPulse;
- список усіх його груп;
- детальну аналітику кожної доступної групи;
- XP, рівні, streak, захисні дні та досягнення;
- рейтинги, тижневі звіти й картки для поширення;
- адміністрування групи без необхідності вводити команди;
- додатковий функціонал у приватному чаті з ботом.

Mini App має виглядати як завершений преміальний продукт, бути повністю адаптованим під Telegram на iOS та Android і використовувати вже наявну базу ChatPulse v3.

## 2. Межі першої версії

До першої версії входять усі екрани, API та сценарії, описані в цій специфікації. Вона не включає платні підписки, Telegram Stars, сторонні соціальні входи, збереження текстів повідомлень, окремий вебсайт поза Telegram або систему створення власних досягнень користувачами.

## 3. Обрана архітектура

Mini App розміщується в тому самому репозиторії й тому самому Google Cloud Run сервісі, що й Python-бот.

```text
ChatPulse/
├── app/                         # FastAPI, aiogram, SQLAlchemy
│   ├── api/miniapp/             # Mini App API
│   ├── repositories/            # запити до бази
│   └── services/                # auth, cards, notifications
├── miniapp/                     # React + TypeScript + Vite
│   ├── src/api/
│   ├── src/components/
│   ├── src/features/
│   ├── src/pages/
│   └── src/telegram/
└── Dockerfile                   # multi-stage Node + Python build
```

Vite збирає React-фронтенд у статичні файли. Dockerfile копіює збірку в Python image. FastAPI віддає Mini App на `/miniapp` і API на `/api/miniapp/v1`.

Переваги рішення:

- один домен і один production deployment;
- відсутність окремого CORS-контуру;
- спільний доступ до Supabase;
- простіша Telegram-авторизація;
- одна система логування, тестування та CI.

## 4. Telegram-авторизація та доступ

Кожен API-запит Mini App передає оригінальний Telegram `initData` у заголовку `X-Telegram-Init-Data`.

Backend:

1. перевіряє HMAC-підпис за токеном бота;
2. перевіряє `auth_date` і відхиляє прострочені дані;
3. дістає Telegram user ID тільки з перевіреного `initData`;
4. не приймає user ID із query, body або localStorage як джерело ідентичності.

У production відкриття `/miniapp` поза Telegram показує екран «Відкрийте через Telegram». Локальний режим розробки може використовувати тестового користувача лише через окрему development-конфігурацію, яка не активується у production.

### Доступ до груп

Користувач бачить тільки групи, де існує його `group_members` запис. При відкритті детальної групової панелі backend додатково перевіряє актуальне членство через Telegram Bot API `getChatMember`. Якщо Telegram не підтверджує членство, доступ до детальної інформації не надається.

### Адміністраторські дії

Перед кожною зміною налаштувань або скиданням статистики backend повторно викликає `getChatMember` і дозволяє запис лише ролям `administrator` або `creator`. Відповідь фронтенду про роль ніколи не є доказом прав.

## 5. Навігація Mini App

Нижня навігація містить п’ять розділів:

1. **Головна**
2. **Групи**
3. **Рейтинг**
4. **Досягнення**
5. **Профіль**

Навігація залишається видимою на основних екранах. Детальна панель групи відкривається поверх основної структури й має Telegram Back Button.

## 6. Головна сторінка

Головна сторінка є персональним хабом користувача.

### Верхній блок

- Telegram-аватар або ініціали;
- ім’я та username;
- глобальний рівень ChatPulse;
- назва поточного tier;
- progress bar до наступного рівня;
- поточний і необхідний XP;
- глобальне місце користувача без публікації повного глобального рейтингу.

### Швидкі показники

- XP за сьогодні;
- поточний streak;
- рекордний streak;
- доступні захисні дні;
- кількість груп;
- сумарна активність за 7 днів.

### Графік активності

Інтерактивний SVG-графік показує глобальний XP або активність за останні 7 днів. Перемикання між показниками не перезавантажує сторінку.

### Останні досягнення

Виводяться до трьох останніх розблокованих досягнень із рідкістю, датою та групою, де їх було отримано.

### Мої групи

Горизонтальний список найактивніших груп показує:

- назву;
- Telegram-фото групи, якщо його вдалося безпечно отримати;
- fallback-аватар з ініціалами;
- рівень користувача в групі;
- місце в рейтингу;
- XP за 7 днів;
- відсоткову зміну проти попередніх 7 днів.

### Картка профілю

Кнопка «Поділитися профілем» генерує PNG-картку з рівнем, XP, streak, tier та ключовими досягненнями. Картка генерується на backend без збереження файлу в базі.

## 7. Розділ «Групи»

Сторінка містить усі доступні користувачу групи.

### Пошук і сортування

- пошук за назвою;
- сортування за активністю, рівнем, XP або останньою активністю;
- фільтр «Я адміністратор».

### Картка групи

- фото або fallback-аватар;
- назва;
- груповий рівень користувача;
- поточний streak;
- місце в рейтингу;
- повідомлення й реакції за 7 днів;
- тренд активності;
- admin badge, якщо роль підтверджено Telegram.

## 8. Детальна панель групи

### Overview

- загальна кількість повідомлень;
- відповіді;
- медіа;
- реакції;
- активні учасники;
- зміна кожного показника проти попереднього періоду.

Періоди: 7 днів, 30 днів і весь час. Для порівняння 7 та 30 днів використовуються попередні періоди такої самої довжини.

### Графік і теплова карта

- денний графік активності;
- теплова карта за днями та годинами;
- перемикання метрики: повідомлення, реакції, відповіді або XP.

Теплова карта будується тільки з агрегованих лічильників. Тексти повідомлень не потрібні й не зберігаються.

### Особистий прогрес у групі

- XP і рівень;
- progress bar;
- місце в рейтингу;
- streak;
- рекордний streak;
- захисні дні;
- статистика за поточний період.

### Топ учасників

- топ за XP;
- повідомленнями;
- реакціями;
- відповідями;
- streak.

Показуються максимум 50 учасників із пагінацією. Поточний користувач завжди бачить власну позицію, навіть якщо вона нижче топ-50.

### Повідомлення тижня

Показуються:

- автор;
- кількість реакцій;
- дата;
- кнопка переходу до Telegram-повідомлення.

Текст повідомлення не завантажується і не зберігається.

### Номінації

Відображаються поточні тижневі номінації ChatPulse, зокрема балакун тижня, майстер відповідей, улюбленець групи, фотограф, голос групи, нічний житель і рання пташка.

## 9. Рейтинг

Рейтинг працює у двох контекстах:

- вибрана група;
- короткий персональний глобальний результат.

Повний глобальний список користувачів не публікується. У глобальному режимі користувач бачить лише власне місце, percentile і найближчих сусідів без зайвих персональних даних.

Фільтри групового рейтингу:

- XP;
- повідомлення;
- реакції;
- відповіді;
- streak;
- тиждень;
- місяць;
- весь час.

Перші три місця мають окремі podium-картки. Решта відображаються компактним списком.

## 10. Досягнення

Досягнення групуються за категоріями:

- активність;
- реакції;
- streak;
- рівні;
- медіа;
- час активності;
- секретні.

Кожне досягнення містить:

- назву;
- опис;
- іконку;
- рідкість;
- стан `locked` або `unlocked`;
- числовий прогрес;
- дату отримання;
- групу отримання, якщо воно групове.

Секретне досягнення до розблокування показується як прихована картка без умов. Прогрес для звичайних заблокованих досягнень обчислюється з наявних лічильників, а не з текстів повідомлень.

## 11. Профіль

Профіль містить:

- глобальний рівень і tier;
- сумарний XP;
- прогрес до наступного рівня;
- глобальне місце;
- кількість груп;
- поточні й рекордні streak;
- загальну статистику;
- історію останніх досягнень;
- налаштування приватних сповіщень;
- кнопку генерації PNG-картки.

Налаштування сповіщень:

- важливі досягнення;
- глобальний level-up;
- нагадування про streak;
- щотижневий особистий підсумок.

## 12. Щотижневі звіти та історія

Запланований звіт створює `weekly_report_snapshot` із числовим payload:

- підсумкові лічильники;
- порівняння;
- номінації;
- популярна реакція;
- message ID повідомлення тижня;
- автор повідомлення;
- кількість реакцій;
- тема картки;
- дата створення.

Текст повідомлення не входить у snapshot. PNG не зберігається в базі: він повторно генерується з snapshot на запит.

Mini App показує історію до 52 останніх звітів групи. Користувач може:

- переглянути картку;
- перемкнути тему лише для локального preview;
- завантажити картку;
- поширити її через Telegram;
- перейти до повідомлення тижня, якщо воно ще доступне.

## 13. Адмінська панель

Адмінська панель доступна тільки після поточної Telegram-перевірки прав.

Налаштування:

- увімкнення щотижневих звітів;
- день тижня;
- точний час `HH:MM`;
- часовий пояс;
- тема `Dark Pulse`, `Telegram Wave` або `Clean Light`;
- збір повідомлень;
- збір медіа;
- збір відповідей;
- збір реакцій;
- пауза збору статистики.

Для скидання статистики користувач має ввести слово `СКИНУТИ`. Backend ще раз перевіряє admin-роль перед транзакцією. Налаштування групи залишаються, статистичні та gamification-дані групи видаляються.

## 14. Приватний чат із ботом

### `/start`

Показує короткий опис, основну кнопку `Відкрити ChatPulse` типу `web_app` та кнопку додавання бота до групи.

### Menu Button

Бот програмно встановлює Telegram Menu Button, який відкриває `/miniapp`.

### Команди

- `/open` — кнопка відкриття Mini App;
- `/profile` — короткий глобальний профіль;
- `/groups` — список доступних груп із Web App-кнопкою;
- `/achievements` — останні досягнення;
- `/help` — приватна інструкція.

### Приватні повідомлення

Бот може надсилати:

- глобальний level-up;
- важливе глобальне досягнення;
- нагадування про streak;
- щотижневий особистий підсумок.

Повідомлення надсилаються лише користувачам, які запустили приватний чат і не вимкнули відповідну категорію.

### Нагадування про streak

Раз на локальний день групи, близько 20:00, бот перевіряє користувачів із активним streak і менш ніж 10 XP за день.

- якщо є захисні дні, повідомлення попереджає, що буде витрачено один захисний день;
- якщо захисних днів немає, повідомлення попереджає про втрату streak;
- максимум одне нагадування на користувача для однієї групи на день;
- помилка відправлення не блокує інші scheduled tasks.

Наявний Cloud Scheduler кожні 5 хвилин продовжує викликати сумісний внутрішній endpoint. Endpoint обробляє і щотижневі звіти, і нагадування.

## 15. API v1

Основні read endpoints:

```text
GET  /api/miniapp/v1/bootstrap
GET  /api/miniapp/v1/profile
GET  /api/miniapp/v1/groups
GET  /api/miniapp/v1/groups/{chat_id}/overview
GET  /api/miniapp/v1/groups/{chat_id}/activity
GET  /api/miniapp/v1/groups/{chat_id}/leaderboard
GET  /api/miniapp/v1/groups/{chat_id}/achievements
GET  /api/miniapp/v1/groups/{chat_id}/reports
GET  /api/miniapp/v1/groups/{chat_id}/reports/{snapshot_id}/card
GET  /api/miniapp/v1/profile/card
```

Write endpoints:

```text
PATCH /api/miniapp/v1/profile/notifications
PATCH /api/miniapp/v1/groups/{chat_id}/settings
POST  /api/miniapp/v1/groups/{chat_id}/reset
```

Усі endpoints використовують перевірений Telegram initData. Group endpoints перевіряють доступ до групи. Write endpoints додатково перевіряють admin-роль.

API повертає стабільні JSON DTO, а не ORM-моделі. Помилки мають структурований формат:

```json
{
  "error": {
    "code": "GROUP_ACCESS_DENIED",
    "message": "Немає доступу до цієї групи"
  }
}
```

## 16. Нові дані в Supabase

### `weekly_report_snapshots`

Зберігає числові payload звітів і посилання на message ID без тексту.

### `user_notification_settings`

Зберігає opt-in/opt-out для приватних повідомлень і факт запуску приватного бота.

### `notification_events`

Зберігає dedup key, тип, користувача, групу, локальну дату й статус відправлення. Це запобігає повторним level-up, achievement і streak повідомленням.

### Додаткові агрегати

Для теплової карти додаються агреговані погодинні лічильники за групою, користувачем, датою та годиною. У таблиці немає текстів, captions або media content.

Усі нові таблиці отримують індекси для user/group/date queries та RLS із забороною прямого анонімного доступу. Mini App працює через backend, а не напряму з Supabase client.

## 17. Візуальний стиль

Основний стиль:

- графітовий фон;
- синьо-фіолетове glow-світіння;

Анімації:

- плавне заповнення XP progress;

## 18. Компонентні межі

### Backend

- `MiniAppAuthService` — тільки перевірка initData;
- `MiniAppAccessService` — членство й admin-права;
- `MiniAppRepository` — read-моделі для dashboard;
- `MiniAppSettingsService` — валідовані admin writes;
- `SnapshotService` — створення та читання звітів;
- `NotificationService` — dedup і приватні повідомлення;
- `CardService` — profile/report PNG generation.

### Frontend

- `telegram/bridge` — Telegram SDK wrapper;
- `api/client` — initData header, DTO та error mapping;
- `features/profile` — глобальний профіль;
- `features/groups` — список і group dashboard;
- `features/leaderboard` — рейтинги;
- `features/achievements` — каталог і progress;
- `features/reports` — історія й preview;
- `features/settings` — admin controls;
- `components/ui` — повторно використовувані картки, chips, progress, charts і states.

Жоден великий компонент не має одночасно виконувати API-запити, форматувати domain data й малювати складну UI-структуру.

## 19. Обробка помилок

- прострочений initData: екран повторного відкриття Mini App;
- відкриття поза Telegram: branded instruction screen;
- немає груп: empty state із кнопкою додавання бота;
- недостатньо даних для графіка: зрозумілий порожній стан;
- користувач більше не в групі: access denied без витоку статистики;
- Telegram API недоступний під час admin-write: дія не виконується;
- помилка PNG: показується текстова версія та retry;
- network error: кнопка повтору й збереження поточного route;
- часткова помилка одного dashboard widget не повинна ламати весь екран.

## 20. Приватність і безпека

Незмінні правила:

- тексти повідомлень не зберігаються;
- captions не зберігаються;
- медіафайли не зберігаються;
- initData завжди перевіряється на backend;
- user ID не довіряється з frontend payload;
- admin-права перевіряються в Telegram на кожен write;
- Mini App не підключається напряму до Supabase;
- секрети не потрапляють у frontend bundle;
- логи не містять повного initData, bot token або scheduler secret;
- avatar proxy не дозволяє довільні URL чи file IDs від клієнта.

## 21. Тестування

### Python

- валідний, підроблений і прострочений initData;
- access service для member, left, administrator і creator;
- API не дозволяє підставити чужий user ID;
- group data доступні лише учасникам;
- admin writes відхиляються для звичайного учасника;
- reset видаляє статистику, але залишає settings;
