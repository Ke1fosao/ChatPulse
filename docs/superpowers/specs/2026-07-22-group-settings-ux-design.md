# ChatPulse Group Settings UX Design

## Goal

Зробити керування групою однаково зручним у Telegram-боті та Mini App, не послаблюючи перевірку прав адміністратора.

## Telegram group settings

Команда `/settings` відкриває одне головне повідомлення. Усі переходи працюють через `edit_text`, тому бот не засмічує групу новими повідомленнями.

Головне меню містить п’ять розділів:

- **Звіти** — увімкнення звітів, день, час і часовий пояс;
- **Збір даних** — повідомлення, медіа, відповіді й реакції;
- **Оформлення** — тема картки;
- **Стан бота** — пауза або відновлення статистики;
- **Небезпечні дії** — скидання статистики з окремим підтвердженням.

Кожен дочірній екран має кнопку `← Назад`. Перемикачі застосовуються миттєво, після чого те саме повідомлення оновлюється. Час звіту вибирається кнопками з кроком 30 хвилин і не потребує команди `/setreporttime`; команда залишається як сумісний запасний спосіб.

Кожен callback повторно викликає `getChatMember`. Доступ мають лише `administrator`, `creator` і глобальний owner ChatPulse. При втраті прав подальші натискання блокуються.

## Mini App

У списку груп адміністратор бачить помітний бейдж `Ви адміністратор` і короткий підпис про доступ до керування.

На сторінці групи:

- у hero-блоці є чіткий admin-status;
- кнопка `Керувати групою` відкриває окремий повноекранний settings-flow;
- звичайний учасник не бачить кнопки й не може викликати admin API;
- сервер повторно перевіряє актуальний статус через Telegram перед кожним PATCH або reset.

Налаштування згруповані у секції `Звіти`, `Збір даних`, `Оформлення`, `Стан` і `Небезпечна зона`. Кожна зміна зберігається миттєво. Під час запиту control блокується, після успіху показується `Збережено`, при помилці значення повертається до серверного стану.

## Data and API

Нових таблиць не потрібно. Використовуються існуючі `GroupSettings`, `/api/miniapp/v1/groups/{chat_id}/settings` і `/reset`.

`GET /groups` і `GET /groups/{chat_id}` мають віддавати актуальний `is_admin`, отриманий через live Telegram-перевірку, а не лише з історичних даних бази.

## Testing

- unit tests для клавіатур і текстів Telegram settings;
- router tests: навігація редагує одне повідомлення, back повертає головний екран, non-admin отримує alert;
- frontend tests: admin badge, окремий settings screen, autosave success/error rollback;
- API tests: PATCH/reset відхиляються для non-admin;
- повний CI: Ruff, pytest, Vitest, TypeScript, Vite і Docker build.
