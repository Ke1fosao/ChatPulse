# ChatPulse Owner Panel

Owner Panel — окрема керувальна система всередині того самого Telegram Mini App bundle.

- Mini App: `/miniapp`
- Owner Panel: `/miniapp/owner`
- Owner API: `/api/owner/v1`

## Доступ власника

Панель не захищається лише прихованим URL. На кожному запиті backend:

1. перевіряє підпис Telegram `initData` через HMAC;
2. перевіряє строк дії Telegram-сесії;
3. бере Telegram user ID тільки з перевіреного payload;
4. звіряє ID із singleton-записом `bot_owner`;
5. відхиляє всіх інших користувачів кодом `403`.

Після одноразового `/claimadmin` username більше не бере участі в авторизації. Роль власника не можна передати, видалити чи змінити через Owner Panel. VIP-роль також не дає доступу до owner API.

## Можливості Owner Panel

- загальна статистика користувачів, груп, VIP і повідомлень за сім днів;
- пошук користувачів за ім’ям, username або Telegram ID;
- фільтри активних VIP і звичайних користувачів;
- видача безстрокового VIP або VIP до конкретної дати;
- відкликання VIP із обов’язковою причиною;
- пауза групи та керування щотижневими звітами;
- журнал усіх owner-дій.

Видача, відкликання VIP та системні зміни потребують точного підтвердження в API payload. Owner ID завжди визначає сервер, а не frontend.

## VIP-клієнт

VIP отримує весь набір premium entitlements через централізований `app/services/entitlements.py`:

- розширена історія та порівняння;
- premium-теми звітів;
- CSV/PDF-експорт;
- розширені сповіщення;
- AI insights;
- premium profile card;
- усі майбутні функції, що перевіряють `premium.all`.

VIP не отримує:

- доступ до Owner Panel;
- можливість видавати ролі;
- owner-права;
- автоматичні права адміністратора Telegram-групи.

Строковий VIP автоматично стає неактивним після `expires_at`. Безстроковий VIP діє, поки власник його явно не відкличе.

## База даних

Міграція `20260722110000_add_owner_panel_vip.sql` створює:

- `vip_grants` — поточний VIP-стан користувача та історію видачі/відкликання;
- `owner_audit_log` — append-only журнал керувальних дій.

Для обох таблиць увімкнено RLS. Доступ ролям `anon` і `authenticated` відкликаний; таблиці використовує тільки backend через production database connection.

## Приватність

Owner Panel не показує і не записує тексти повідомлень, captions або файли. Доступні лише Telegram profile fields, технічні ID, агреговані лічильники, налаштування груп і системний аудит.
